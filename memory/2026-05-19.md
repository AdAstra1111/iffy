
## IFFY Core Principle: Deterministic Writing (established May 19, 2026)

The AI must never hallucinate or invent story facts. Every character detail, world rule, plot point, relationship dynamic, and dramatic beat must trace directly back to source documents in the upstream ladder.

**Enforcement:**
- If a fact doesn't exist in an upstream doc, the AI must flag it as missing — not invent it
- `beat_sequential` strategy for feature_script is the model: each scene constrained to beat sheet facts + reasonable execution (dialogue, action, setting)
- No new story facts at any stage
- Applies across all formats (film, series, vertical-drama, documentary, etc.)

**Tradeoffs:**
- Slower sequential generation > fast hallucinated output
- Quality of output is paramount — never sacrifice for speed
- Chunk persistence (commit 9490e0f) mitigates speed cost via crash recovery

## Operating Protocol: Oracle ↔ ChatGPT Design Pipeline (established)

For complex work beyond simple fixes, follow this standard sequence:
1. Oracle researches codebase, formulates comprehensive design doc
2. User takes package to ChatGPT for back-and-forth design consultation
3. ChatGPT provides critique + alternatives
4. User returns output to Oracle for integration
5. Oracle updates design doc + Architect task
6. Chain executes with ChatGPT feedback baked in

This is documented in docs/DETERMINISTIC_WRITING_PRINCIPLE.md and docs/beat_sequential_strategy_proposal.md. The beat_sequential feature was the first use of this protocol and produced a significantly stronger architecture.
